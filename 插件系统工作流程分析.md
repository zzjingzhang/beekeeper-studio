## 1. 插件声明 Shell View 并通过菜单创建 Tab 的流程

### 1.1 插件 Manifest 声明

插件通过 `manifest.json` 声明其提供的视图和菜单项。支持两种版本的 manifest 格式：

**Manifest V1 格式**：

```typescript
// types.ts#L139-L144
export type ManifestV1 = Omit<ManifestV0, "manifestVersion" | "capabilities"> & {
  manifestVersion: 1;
  capabilities: Omit<ManifestV0["capabilities"], "views"> & {
    views: PluginView[];
  }
}
```

**PluginView 定义**：

```typescript
// types.ts#L18-L27
export type PluginView = {
  id: string;           // 视图 ID
  name: string;         // 视图名称，用于 UI 显示
  type: `${TabType}-tab`;  // "shell-tab" 或 "base-tab"
  entry: string;        // HTML 入口文件，如 index.html
};
```

**菜单项声明**：

```typescript
// types.ts#L52-L76
export interface PluginMenuItem {
  command: string;      // 菜单命令 ID
  name: string;         // 菜单显示名称
  view: string;         // 关联的视图 ID
  placement: PluginMenuItemPlacement | PluginMenuItemPlacement[];
}
```

## 3. 如何创建 Plugin-Shell Tab

### 3.1 点击下拉菜单项触发创建

[CoreTabs.vue#L670-L695](file:///c:/Users/10244/Desktop/0508-under/beekeeper-studio/apps/studio/src/components/CoreTabs.vue#L670-L695)

```typescript
async createTab(config: TabTypeConfig.Config) {
  if (config.type === "query") {
    this.createQuery()
  } else if (config.type === "shell") {
    this.createShell()
  } else if (config.type === "plugin-shell" || config.type === "plugin-base") {
    // 生成唯一标题
    let tNum = 0;
    let title = config.name;
    do {
      tNum = tNum + 1;
      title = `${config.name} #${tNum}`;
    } while (this.tabItems.filter((t) => t.title === title).length > 0);

    const tab = {
      tabType: config.type,
      title,
      unsavedChanges: false,
      context: {
        pluginId: config.pluginId,
        pluginTabTypeId: config.pluginTabTypeId,
        command: config.menuItem?.command,
        params: config.menuItem?.params,
      },
    } as TransportPluginTab;
    await this.addTab(tab)
  }
}
```

### 5.1 消息通信架构


```typescript
private handleMessage(event: MessageEvent) {
  const view = this.viewInstances.find(
    ({ iframe }) => iframe.contentWindow === event.source
  );
  const source = view?.iframe;

  if (source) {
    if (event.data.id !== undefined) {
      // 请求-响应模式
      this.handleViewRequest({ ...event.data }, source);
    } else {
      // 通知模式
      this.handleViewNotification(source, { ...event.data });
    }
  }
}
```

## 总结

### 完整流程图

```
插件安装
    ↓
PluginManager 扫描 manifest
    ↓
WebPluginManager 加载插件
    ├─ 注册 TabTypeConfig 到 Vuex
    └─ 注册菜单项 (newTabDropdown)
    ↓
CoreTabs 渲染下拉菜单 (newTabDropdownItems)
    ↓
用户点击菜单项
    ↓
createTab() 创建 TransportPluginTab
    ├─ tabType: "plugin-shell" 或 "plugin-base"
    └─ context: { pluginId, pluginTabTypeId, command, params }
    ↓
TabModule.add() 保存到数据库并添加到状态
    ↓
TabPluginShell/TabPluginBase 渲染
    ├─ 检查 Community 限制 (bks- 前缀)
    ├─ 渲染 IsolatedPluginView (iframe)
    └─ 注册 onRequest 监听器
    ↓
插件 iframe 发送消息 (postMessage)
    ↓
WebPluginLoader.handleMessage()
    ├─ runQuery → PluginStoreService.runQuery() → Tab 更新结果
    ├─ setTabTitle → Tab 组件修改 title 并保存
    └─ setViewState/getViewState → Tab 组件操作 context.state 并保存
```
